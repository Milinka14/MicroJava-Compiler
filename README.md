A compiler for the **MicroJava (MJ)** programming language, implemented in Java. The compiler performs lexical analysis, parsing, semantic analysis, and bytecode code generation, producing a `.obj` file that can be run on the MicroJava virtual machine.

---

## 📁 Project Structure

```
MJCompiler/
├── src/
│   └── rs/ac/bg/etf/pp1/
│       ├── Yylex.java          # Generated lexer (JFlex)
│       ├── MJParser.java       # Generated parser (CUP)
│       ├── sym.java            # Token symbols (generated)
│       ├── SemAnalyzer.java    # Semantic analysis (visitor)
│       ├── CodeGenerator.java  # Bytecode code generation (visitor)
│       ├── ast/                # AST node classes (generated by CUP)
│       └── util/               # Utility/helper classes
├── spec/
│   ├── mjlexer.lex             # JFlex lexer specification
│   └── mjparser.cup            # CUP parser/grammar specification
├── lib/
│   ├── JFlex.jar               # Lexer generator
│   ├── cup_v10k.jar            # Parser generator
│   ├── mj-runtime-1.1.jar      # MicroJava virtual machine runtime
│   └── log4j-1.2.17.jar        # Logging library
├── test/
│   ├── program.mj              # Example MicroJava source file
│   ├── program.obj             # Compiled output bytecode
│   └── ulaz.txt                # Standard input for the program at runtime
├── config/                     # Logging configuration
├── logs/                       # Compiler log output
└── build.xml                   # Apache Ant build script
```

## ⚙️ Prerequisites

- **Java JDK** 8 or higher
- **Apache Ant** (to use the `build.xml` build script)
  - Install on Ubuntu/Debian: `sudo apt install ant`
  - Install on Windows: [Download Ant](https://ant.apache.org/bindownload.cgi) and add to `PATH`
- All required JARs are already included in `MJCompiler/lib/`

## 🚀 How to Build and Run

All commands should be run from inside the `MJCompiler/` directory:

```bash
cd MJCompiler
```

### Step 1 — Generate Lexer & Parser, then Compile

This single command runs the full build pipeline:
- Deletes previously generated files
- Generates the lexer from `spec/mjlexer.lex` using JFlex
- Generates the parser and AST from `spec/mjparser.cup` using CUP
- Fixes package references
- Compiles all Java source files

```bash
ant compile
```

### Step 2 — Run the Compiled Program

Once you have a compiled `.obj` file (e.g. `test/program.obj`), run it on the MicroJava VM:

```bash
ant runObj
```

> This reads input from `test/ulaz.txt` and runs `test/program.obj`.

### Step 3 (Optional) — Disassemble the Bytecode

To inspect the generated bytecode:

```bash
ant disasm
```

### Step 4 (Optional) — Run in Debug Mode

```bash
ant debugObj
```

## 🔧 Build Targets Summary

| Target       | Description                                                  |
|--------------|--------------------------------------------------------------|
| `delete`     | Removes previously generated lexer/parser/AST files         |
| `lexerGen`   | Generates `Yylex.java` from `spec/mjlexer.lex` using JFlex  |
| `parserGen`  | Generates `MJParser.java` and AST from `spec/mjparser.cup`  |
| `repackage`  | Fixes generated package references                          |
| `compile`    | Runs the full build pipeline (delete → lexerGen → parserGen → repackage → javac) |
| `disasm`     | Disassembles `test/program.obj`                             |
| `debugObj`   | Runs `test/program.obj` in debug mode                       |
| `runObj`     | Runs `test/program.obj` on the MicroJava VM                 |

## ✍️ Writing a MicroJava Program

MicroJava source files use the `.mj` extension. Place your source file in the `test/` folder. Here's an example (`test/program.mj`):

```
program test
{
    void x(int a, int b) {
        switch (a) {
            case 0: break;
            case 1: print('a');
            case 2: {
                switch(b) {
                    case 5: return;
                }
            }
        }
    }

    void main() int i;
    {
        for (i = 0; i < 4; i++) {
            switch(i) {
                case 0: print(i); break;
                case 1: continue;
                case 2: {
                    switch (i) {
                        case 0: break;
                        case 2: print(i);
                        case 3: continue;
                    }
                }
                case 3: {
                    for (;;) break;
                }
            }
        }
        x(1, 5);
    }
}
```

Every MicroJava program must have a `void main()` method as the entry point.

---

## 🧪 Providing Runtime Input

If your program reads from standard input, put the input values in `test/ulaz.txt` (one value per line). The Ant targets (`runObj`, `debugObj`) automatically pipe this file as stdin.

## 🗂️ How It Works

```
source.mj
   │
   ▼
[Lexer — Yylex.java]         (generated from spec/mjlexer.lex via JFlex)
   │  token stream
   ▼
[Parser — MJParser.java]     (generated from spec/mjparser.cup via CUP)
   │  Abstract Syntax Tree (AST)
   ▼
[Semantic Analyzer — SemAnalyzer.java]   (visitor over AST)
   │  annotated/validated AST
   ▼
[Code Generation — CodeGenerator.java]   (visitor over AST)
   │  JVM Bytecode
   ▼
